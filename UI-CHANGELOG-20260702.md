## 一、修改背景

AIGC-MAS 是一个多智能体 AI 视频制作服务平台，深度垂直服务于三个业务场景：
1. AIGC 短剧生产/承制
2. AIGC 企业视频宣传与营销视频承制
3. AIGC 地方文旅宣传承制

### 发现的问题

1. **核心视频制作功能缺失**：VideoWorkbench.tsx 虽然有 82KB 的节点式工作流编辑器框架，但缺少参考系统（麦预演/麦视频/星流演）中的关键功能环节：预演图生成、分镜拆解弹窗、创作模式选择入口、空白画布引导状态等
2. **UI 风格不统一**：HTML 原型使用品牌名 VideoAI，React 应用使用 AIGC-MAS；数据统计数字两套不一致；导航项和页面结构存在差异
3. **PRD 缺少关键流程描述**：3.4 节视频制作工作台缺少渐进式生成流程、预演图节点、分镜拆解等核心功能的规格定义

### 参考系统

修改参考了 `maiyuyan/` 目录下的 9 张系统截图（麦预演、麦视频、麦序演、星流演），提取了完整的视频制作工作流 UI 交互模式。

## 二、修改内容详解

### 2.1 PRD 补充（PRD.md）

在 3.4.9 节之后、3.5 节之前，新增 5 个小节：

| 新增章节 | 内容概要 |
|---------|---------|
| 3.4.10 工作台入口 — 创作模式选择 | 6种创作模式卡片（短剧/电影/电视剧/电商/手搓/漫游），2x3网格布局，渐变遮罩缩略图，"立即创作"入口 |
| 3.4.11 空白画布引导状态 | 空白项目引导文字"右击画布 添加短视频脚本 或 分镜镜头"，点阵网格背景（24px间距） |
| 3.4.12 渐进式视频生成流程 | 完整5步链路（脚本→预演图→分镜拆解→视频生成→任务管理），积分消耗对比表，设计原则 |
| 3.4.13 预演图节点 | 节点功能描述：AI生成预演图（3.8积分/张）、本地上传/历史复用、生成预览流程 |
| 3.4.14 分镜拆解弹窗 | 弹窗UI规格：镜头数量控制器（1-10）、步骤指示器、操作按钮 |

**核心设计决策**：渐进式生成流程是平台的业务差异化能力。用户不必一次性配置所有参数生成最终视频，而是通过低成本的预演图确认视觉方向后再投入积分生成高质量视频，降低试错成本。

## 三、修改后的视频制作完整工作流

```
用户登录 → 仪表盘 → 选择创作模式（6选1）
    → 进入空白画布工作台（引导提示）
        → 右键/"+"添加脚本节点 → 输入/生成剧本文案
            → 添加预演图节点 → 上传或AI生成预演图（3.8积分）
                → 分镜拆解弹窗 → 选择镜头数量（1-10）→ 生成分镜
                    → 视频生成设置（比例/分辨率/模型/时长/风格）
                        → 提交生成任务 → 任务管理（进度/状态/积分）
                            → 预览/下载/重新生成
```

## 五、遗留事项与后续建议

### 需要开发团队跟进的事项

1. **后端 API 对接**：当前预演图生成和分镜拆解为前端模拟实现，需要对接 seedanceService 和视频生成 API
2. **积分系统集成**：预演图生成（3.8积分/张）和视频生成（8.3积分/秒）的积分扣减需要对接支付系统
3. **资产库集成**：预演图节点的"本地上传"和"历史复用"需要对接 Assets.tsx 页面的资产管理系统
4. **AI 智能体协同**：脚本生成环节需要对接 AgentEngine 中的编剧智能体
5. **创作模式模板**：6种创作模式需要各自预配置不同的智能体组合和工作流模板

### UI 改进建议

1. HTML 原型（UI设计/目录）与 React 应用仍为两套独立代码，长期应考虑统一为一套 React 代码
2. 建议将 6 种创作模式的缩略图替换为实际的场景预览图（当前使用 CSS 渐变占位）
3. 分镜拆解后的镜头编辑界面需要进一步细化（当前弹窗仅完成镜头数量选择）
